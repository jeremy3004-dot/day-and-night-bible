# Plan 01 Summary

## Outcome

EveryBible now has a real audio-only translation path for no-text testing and usage. Users can select an audio-only translation when audio is actually playable (remote or offline), and both reader and daily-scripture presentation logic now explicitly handle the edge case where text is missing and no audio source is available.

## Changes

- Added `src/screens/bible/bibleTranslationModel.ts` and `src/screens/bible/bibleTranslationModel.test.ts` to centralize translation selection eligibility, including the audio-only/no-audio fallback rule.
- Added an audio-only translation profile in `src/constants/translations.ts` (`bsb_audio`) with chapter-based audio metadata and no text.
- Updated `src/screens/bible/BibleBrowserScreen.tsx` to use selection-state rules for translation picking and surface clearer availability states for selectable audio-only translations.
- Updated `src/screens/bible/BibleReaderScreen.tsx` to apply the same selection-state rules in reader translation switching and provide explicit feedback when audio cannot currently play.
- Updated `src/stores/bibleStore.ts` so store-level translation switching allows audio-only translations only when runtime audio capability exists.
- Expanded no-text fallback coverage in `src/services/bible/presentation.test.ts` and `src/services/bible/dailyScripture.test.ts` to assert both:
  - audio available -> audio-first/section audio behavior
  - audio unavailable -> empty fallback behavior
