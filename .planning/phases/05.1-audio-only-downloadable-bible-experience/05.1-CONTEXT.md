# Phase 05.1: Audio-only downloadable Bible experience - Context

**Gathered:** 2026-03-14
**Status:** Ready for planning
**Source:** User request in active session

<domain>
## Phase Boundary

Deliver an in-app path to test and use an audio-only Bible experience where chapter text is unavailable but audio playback/download still works. This phase must stay inside existing Expo/React Native architecture and reuse the current audio manager, reader, and presentation helpers.

Scope includes:
- adding at least one audio-only translation profile for real app testing
- enabling sane selection/download behavior for that translation
- validating no-text reader and daily-scripture states when audio is available

Out of scope:
- new backend services or schema changes
- redesigning the full reader/audio player UI
- broad translation marketplace work
</domain>

<decisions>
## Implementation Decisions

### Locked Decisions
- Add an explicit audio-only translation option in the app to support manual QA for "audio exists, text does not".
- Audio-only mode must still support offline audio downloads through existing download flows.
- Reader surfaces must avoid generic missing-content messaging when no text exists but audio is playable.
- Keep existing text-first translation behavior unchanged for current users.

### Claude's Discretion
- Exact translation identifier/name used for the audio-only profile.
- Whether translation selection is enabled pre-download or gated on first audio download.
- Wording for small UI cues that communicate audio-only/no-text state.
</decisions>

<specifics>
## Specific Ideas

- Reuse existing `hasText`, `hasAudio`, `audioGranularity`, and `downloadedAudioBooks` contracts.
- Keep decision logic in small pure model/helpers with tests, then consume from screens.
- Prioritize deterministic unit coverage around translation selection eligibility and presentation mode decisions.
</specifics>

<deferred>
## Deferred Ideas

- Downloadable text-translation marketplace (`MULTI-01`) beyond this targeted no-text audio test path.
- Rich per-translation onboarding/education flows.
- New audio provider integrations.
</deferred>
