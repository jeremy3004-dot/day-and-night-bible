# Plan 06-01: Seeded Bible Data Bootstrap

## Objective

Replace the runtime JSON Bible seed path with a bundled indexed SQLite asset that opens faster, recovers from broken legacy local data, and gives offline search a stronger foundation.

## Scope

- Add a reproducible script that builds a seeded Bible SQLite asset from the existing processed JSON source.
- Bundle the generated database with the app.
- Change the Bible database/bootstrap services to import the bundled asset, verify readiness, and recover from partial or stale local databases.
- Prefer indexed local search and keep a graceful fallback path.
- Add regression tests around readiness and query normalization rules.

## Out Of Scope

- Multi-translation download infrastructure
- Reader UI redesign
- Home dashboard redesign

## Implementation Steps

1. Add a build script for the seeded Bible database asset with schema/version metadata and an FTS index.
2. Commit the generated database asset and ensure Expo bundles it as a native asset.
3. Replace runtime JSON import logic in the Bible services with bundled-database import, verification, and overwrite recovery.
4. Switch search to indexed query generation with a fallback for unsupported or malformed searches.
5. Add and run targeted tests plus typecheck/lint/database verification commands.

## Risks

- Asset bundling can be fragile if the database file is not explicitly included in Expo config.
- FTS query syntax can fail on malformed input; search needs a safe fallback.
- Existing users may already have empty or partially seeded local databases; readiness verification must catch and repair those cases.
