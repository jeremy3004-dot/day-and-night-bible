# Plan 12.1.1-01: Run The Health Sweep And Remediate Evidence-Backed Failures

## Objective

Turn the current post-release health sweep into concrete runtime and backend evidence, then fix the most credible breakages or hang risks without destabilizing the shipped app.

## Scope

- Perform a native iOS smoke test covering startup, Bible navigation, and chapter open behavior.
- Capture any reproduced frontend/runtime issue as a failing test or deterministic repro before implementation.
- Inspect backend SQL/policy definitions and remediate the highest-confidence advisory issues that are safe to fix in-repo.
- Re-run the relevant automated verification and backend checks after remediation.

## Out Of Scope

- Feature redesign work
- Unverified speculative performance refactors
- TestFlight submission

## Verification

```bash
npm run release:verify
npm test
npm run ios -- --simulator "iPhone 16"
```
