**Step 2: Create `scripts/import-ebible-translations.ts`**

A TypeScript script runnable with `npx tsx scripts/import-ebible-translations.ts`. It must:

1. **Accept CLI args:** `--translations <comma-separated-ids>` (e.g., `engwebp,spaRV1909`) or `--languages <comma-separated-codes>` (e.g., `eng,spa,hin,npi`) or `--all` (imports every translation in translations.csv matching the OT=39, NT=27, Redistributable=True, downloadable=True filter).

2. **Parse translations.csv:** Fetch `https://ebible.org/Scriptures/translations.csv`, parse it (use built-in Node.js CSV handling or simple split-by-comma with quote handling), filter to full Bibles (OTbooks=39, NTbooks=27, Redistributable=True, downloadable=True).

3. **For each translation to import:**
   a. **Upsert translation_catalog row** with metadata from CSV (translation_id, name=shortTitle, abbreviation=shortTitle first word, language_code=languageCode, language_name=languageNameInEnglish, license_type from Copyright, source_url=`https://ebible.org/Scriptures/{translationId}`, has_text=true, has_audio=false, is_bundled=false, is_available=true, text_direction from textDirection column).

   b. **Download VPL zip:** `https://ebible.org/Scriptures/{translationId}_vpl.zip`

   c. **Extract `{translationId}_vpl.xml` from the zip** (use `node:zlib` and a zip library, or use the `.txt` fallback format if XML is missing).

   d. **Parse VPL XML:** Each verse is `<v b="GEN" c="1" v="1">verse text</v>`. Use a simple regex or lightweight XML parser. Map the `b` attribute through `EBIBLE_BOOK_MAP`. Skip books not in canonical 66 (deuterocanonical books).

   e. **Bulk upsert into bible_verses:** Batch in chunks of 500 rows using the Supabase JS client with `upsert({ onConflict: 'translation_id,book_id,chapter,verse' })`. Use the SUPABASE_SERVICE_ROLE_KEY env var (bypasses RLS).

   f. **Upsert translation_versions row** with version_number=1, is_current=true, total_verses=count.

4. **Use env vars:**
   - `SUPABASE_URL` (or `EXPO_PUBLIC_SUPABASE_URL`)
   - `SUPABASE_SERVICE_ROLE_KEY` (not the anon key — needed for insert)

5. **Be idempotent:** Use upsert everywhere. Safe to re-run.

6. **Log progress:** `[import] Importing {translationId} ({languageName})... {versesInserted} verses`

7. **Error handling:** If a translation zip fails to download or parse, log a warning and continue with the next translation. Do not abort the entire run.

8. **Add a `--dry-run` flag** that parses CSV, lists translations that would be imported, but does not download or insert.

The script requires `npm install @supabase/supabase-js` (already in package.json) and `npm install adm-zip` (for zip extraction — add as devDependency). Use `npx tsx` to run TypeScript directly.
