<objective>
Swap AsyncStorage for MMKV v2.12.2 across all 7 persisted Zustand stores, add a one-time AsyncStorage-to-MMKV migration helper to preserve existing user data, and update SettingsScreen cache/delete flows to clear MMKV.

Purpose: Eliminate cold-start hydration lag caused by AsyncStorage's async JS bridge. MMKV reads are synchronous via JSI, so stores are ready before the first React render.

Output: All stores use MMKV. Migration helper preserves user reading progress, preferences, and course completions. Dev build required for testing (native module).
</objective>
