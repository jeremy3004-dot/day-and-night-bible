    Create a new migration that does three things in one commit:

    1. Add `chapter_feedback_enabled BOOLEAN NOT NULL DEFAULT FALSE` to `public.user_preferences`.
    2. Create `public.chapter_feedback_submissions` with:
       - `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`
       - `user_id UUID NULL REFERENCES public.profiles(id) ON DELETE SET NULL`
       - `translation_id TEXT NOT NULL`
       - `translation_language TEXT NOT NULL`
       - `interface_language TEXT NOT NULL`
       - `content_language_code TEXT`
       - `content_language_name TEXT`
       - `book_id TEXT NOT NULL`
       - `chapter INTEGER NOT NULL CHECK (chapter >= 1)`
       - `sentiment TEXT NOT NULL CHECK (sentiment IN ('up', 'down'))`
       - `comment TEXT NULL CHECK (comment IS NULL OR char_length(trim(comment)) BETWEEN 1 AND 2000)`
       - `source_screen TEXT NOT NULL DEFAULT 'reader'`
       - `app_platform TEXT`
       - `app_version TEXT`
       - `export_status TEXT NOT NULL DEFAULT 'pending' CHECK (export_status IN ('pending', 'exported', 'failed'))`
       - `exported_at TIMESTAMPTZ`
       - `export_error TEXT`
       - `created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()`
       - `updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()`
    3. Add indexes for the exact query patterns ops will need:
       - `(translation_language, book_id, chapter, created_at DESC)`
       - `(export_status, created_at ASC)`
       - `(user_id, created_at DESC)` where `user_id IS NOT NULL`

    Enable RLS on `chapter_feedback_submissions`, but do not add client-facing select/update policies. The app should submit through the Edge Function, not through direct table writes. Service-role access remains sufficient for the function. Add an `updated_at` trigger using the existing helper.

    - a new `ChapterFeedbackSubmission` interface that mirrors the table

    Extend `src/services/sync/syncService.ts` so synced preferences read and write `chapter_feedback_enabled` alongside the existing reminder and locale fields. This is the flag that keeps new installs and existing installs off by default until a user opts in.

  <action>
    Add a new Supabase Edge Function named `submit-chapter-feedback` with `verify_jwt: true`.

    Function contract:
    - Request body:
      ```json
      {
        "translationId": "bsb",
        "translationLanguage": "English",
        "bookId": "JHN",
        "chapter": 3,
        "sentiment": "up",
        "comment": "The chapter title could be clearer",
        "interfaceLanguage": "en",
        "contentLanguageCode": "en",
        "contentLanguageName": "English",
        "sourceScreen": "reader",
        "appPlatform": "ios",
        "appVersion": "1.0.0"
      }
      ```
    - Validation:
      - reject missing `translationId`, `translationLanguage`, `bookId`, `chapter`, `sentiment`, `interfaceLanguage`
      - reject `chapter < 1`
      - reject comment length > 2000
      - trim comment; blank comment becomes `null`
    - Behavior:
      1. Resolve the authenticated user from the bearer token
      2. Insert the row into `chapter_feedback_submissions` with `export_status='pending'`
      3. Append a spreadsheet row to a tab named after normalized `translationLanguage` (example: `English`, `Spanish`, `Nepali`), creating the tab and header row if missing
      4. On success, set `export_status='exported'` and `exported_at=NOW()`
      5. On spreadsheet failure, keep the row, set `export_status='failed'`, persist `export_error`, and return a degraded-success response so the app can tell the user their feedback was saved but not fully exported

    Use secrets, not literals:
    - `GOOGLE_SHEETS_SPREADSHEET_ID`
    - `GOOGLE_SERVICE_ACCOUNT_EMAIL`
    - `GOOGLE_SERVICE_ACCOUNT_PRIVATE_KEY`

    `docs/chapter-feedback-ops.md` must document:
    - required secrets and how to set them in Supabase
    - spreadsheet tab naming rule
    - required columns and header order
    - how operators find rows stuck in `failed`
    - that Supabase remains the source of truth if Sheets is unavailable
  </action>

  </verify>
