  <action>
    Build the shipped UX as follows:

    Settings:
    - Add a new row in Settings, ideally under reading or a new "Feedback" section, with a `Switch`.
    - Label it clearly as chapter feedback and describe it as optional.
    - When toggled, update `preferences.chapterFeedbackEnabled` and call `syncPreferences().catch(() => {})`, matching existing settings behavior.

    BibleReader:
    - Keep the reader clean. Do not add persistent thumbs to the main reading chrome.
    - Add a `Chapter feedback` action to the existing chapter action sheet only when `chapterFeedbackEnabled` is true.
    - Tapping it opens a modal or sheet with:
      - chapter reference title
      - two mutually exclusive buttons for thumbs up / thumbs down
      - optional multiline text input for suggestions or issues
      - cancel and submit actions
    - Use a small pure model module (`bibleReaderFeedbackModel.ts`) for:
      - validating `sentiment`
      - trimming comment text
      - deriving whether submit should be enabled
      - deciding the degraded-success message when the backend saved but spreadsheet export failed

    Submission:
    - `chapterFeedbackService.ts` should call the `submit-chapter-feedback` Edge Function.
    - Include `translationId`, `translationLanguage`, `bookId`, `chapter`, `sentiment`, `comment`, `interfaceLanguage`, and current content-language preference metadata.
    - On network failure or non-OK response, keep the draft in component state and show retry-safe error UI.
    - On success, clear the draft, close the modal, and show concise confirmation copy.

    Tests:
    - `bibleReaderFeedbackModel.test.ts` covers validation and CTA state
    - `bibleReaderFeedbackSource.test.ts` checks the reader source for the settings-gated action, sentiment buttons, optional multiline field, and submit call wiring
    - Update locale files for en/es/ne/hi at minimum, since those are the app's core surfaced locales
